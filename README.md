    Vad är HTML och dess roll inom frontend? Vad menas med semantiskt html?:
HTML är ett typ av språk för struktur på en webbsida, medan det finns alla möjliga bibliotek för att göra hemsidor så kan dom inte göra mycket till en hemsida om man inte har ett sätt att visa sidan på internet, tänk HTML som skelettet för en webbplats som man sen kan lägga på annat. Semantisk html är när man strukturerar HTML inte med hur den ser ut, utan dess mening och koppling till andra element, alltså man lägger in bland annat sections, labels och inte bara en massa div:s. Detta är för att hjälpa datorerna förstå hur hemsidan ska vara strukturerad för att hjälpa bland annat blinda så man kan ha hjälpverktyg beskriva hur hemsidan är strukturerad.


    Vad är CSS och dess roll inom frontend?:
CSS är ett typ av språk som specificerar hur något ser ut, om HTML är skelettet så är CSS skinnet på en webbsida, alla HTML element har som standard CSS renad inbyggt i dom, om det inte har CSS så vet inte datorn vad den ska visa.


    Vad menas med responsiv design samt med vilka metoder man kan utveckla responsiva sidor?:
Responsiv design är att en webbplats ska anpassa sig efter skärmstorleken av maskinen som laddat in webbsidan, så att man kan så lätt som möjligt se och läsa av allt på webbsidan. Detta kan bli gjort med att när man väljer storlek på olika element väljer man procent eller rem så den automatisk anpassar sig för hur stor skärmen är, också använda sig av @media för att ändra vissa element om det skulle bli opraktisk att bara förminska allt.


    Beskriv några konkreta tekniker enligt WCAG som kan förbättra en webbplats tillgänglighet:
Några exempel är: Alternativtext för att ge en beskrivning av en bild för dom som inte kan se bilden, eller när servern kraschar. Tillräckligt stor färgkontrast så färgblinda kan se skillnad mellan element. HTML struktur, använda sig av “header” “nav” “main” “fotter”, användbart för blinda så dom kan få en bättre beskrivning, lättare att programmera så koden inte blir spagetti indirekt gör det bättre för alla användare.


    Redogör för några av de viktigaste UX/UI-principerna inom frontend?:
Tydlighet och enkelhet: se till att hemsidan är tydlig med vart enda knapptryck ska göra, med en snabb blick borde man kunna få en bra idé vad allt gör. Hitta inte på massa nya symboler och design utan en bra anledning, standard finns av en anledning, när majoriteten av webbsidor har många likheter så hjälper det användare att känna igen sig, navbars är på toppen av hemsidor, meny är ofta en hamburgare, det mesta är strukturerad till höger, osv.

Tillgänglighet: Gör hemsidan användbar för så många som möjligt, som tidigare pratat om, bra struktur i html och beskrivningar för blinda, färgkontrast för färgblinda, sen också generelt ha färg och design som är tydlig och inte ha någon stark färg som inte ser gräslig ut så folk inte hoppar av sidan sekunden dom ser den.

Responsivitet och flexibilitet: Av samma anledning som tillgänglighet att ha så många användare som möjligt, gör så många skärmstorlekar kan använda webbsidan, om man förlorar potentiella kunder från att de inte ens kan använda sidan så sätter man sig upp för misslyckande.
